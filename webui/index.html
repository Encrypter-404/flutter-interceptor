Burp / HTTP Toolkit setup

The tool routes the app's traffic to your PC. Your proxy just needs an invisible listener.
1

Add an invisible proxy listener

Burp → Proxy → Proxy settings → Add. Bind to port 8080, bind to All interfaces (or loopback), and under Request handling tick “Support invisible proxying”. This is required — Flutter traffic arrives transparently with no proxy headers.

2

Match the port here

Set the Proxy PC : port on the Intercept tab to the same port your listener uses. The tool builds an adb reverse tunnel + iptables redirect so the device reaches your PC on that port.

3

Trust isn't needed — TLS is unpinned in-app

The engine patches Flutter's BoringSSL ssl_verify_peer_cert and the Java/Conscrypt trust managers, so you'll see decrypted HTTPS without installing the Burp CA on the device.

4

Intercept the same app you launch

Clicking an app card sets the redirect for that app's uid only. Pick the app you actually want to capture — running a different app won't be routed.

When you hit Stop (or close the window) everything is reverted: hooks detach, iptables rules + adb-reverse are removed, route_localnet is restored, and the frida-server is killed. The device returns to normal.

Crash troubleshooting — what to enable

When an app crashes, the Activity log prints a DIAGNOSIS. This is the cheat-sheet.
A

App closes instantly / stuck at splash → frida-server detection

The app detects frida-server (process name / port 27042) and self-kills. Enable “Stealth (experimental)” (runs frida-server hidden) and keep “Anti-detection” ON. The tool auto-recovers crashes by default.

B

“connection-terminated” in the log → server killed

The frida connection dropped. Enable Stealth; auto-recover reprovisions the server and re-applies patches automatically.

C

Still crashing with Stealth ON → try Attach

Switch Mode → Attach: open the app yourself, let it fully load past its checks, then click it in the tool. Spawn-time checks are the hardest, and attaching after load often survives.

D

You see “kept app alive (blocked …)” then it still dies

That's “Anti-detection” working — it blocked the app's exit/kill/tgkill/raise self-kills. If it dies anyway, the app escalated to a deliberate SIGSEGV crash that can't be blocked generically — that app needs a per-app anti-tamper patch.

Recommended defaults for stubborn apps: SSL ✓ · Root ✓ · Anti-detection ✓ · Auto-recover ✓ · Stealth ✓, and if it still won't stay up, Mode → Attach.
